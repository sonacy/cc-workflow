# Implementation Plan: Workflow v2

> Generated by /ww:plan on 2026-03-25

## Overview

Major workflow improvements: /revert command, /archive command, sequential /implement with auto-PR, and interactive code review in /done.

## Steps

### Step 1: Create /revert command

- **Description**: New command with 3 modes: `/revert plan` (reset to before plan, discard everything), `/revert implement` (reset to plan commit, keep plan docs), `/revert debug` (reset to last implement commit, discard debug fixes). Uses `git reset --soft` strategy. Guards against reverting to a state you haven't reached yet.
- **Files**: commands/revert.md
- **Complexity**: Medium
- **Depends on**: none

### Step 2: Create /archive command

- **Description**: Drop the current feature entirely. Ask user for reason, write archive doc to `.claude/workflow/archive/<slug>-archived.md`, then `git reset --hard` to default branch. Delete state file.
- **Files**: commands/archive.md
- **Complexity**: Low
- **Depends on**: none

### Step 3: Rewrite /implement to run all steps sequentially + auto-PR

- **Description**: Instead of one step per invocation, run ALL pending steps sequentially. Show progress as each step completes. After all steps: run lint, tests, build. Fix errors automatically. Then detect platform (check git remote for github.com → `gh`, gitlab.com → `glab`, else ask user) and create PR/MR.
- **Files**: commands/implement.md
- **Complexity**: High
- **Depends on**: none

### Step 4: Rewrite /done with interactive code review

- **Description**: /done always runs code review first. Present findings to user. WAIT for user to approve/reject each finding. Apply approved fixes. Re-run lint+tests after fixes. Only push after clean. Then merge check + switch to default branch.
- **Files**: commands/done.md
- **Complexity**: Medium
- **Depends on**: none

### Step 5: Update install.sh, CLAUDE.md, README.md

- **Description**: Add /revert and /archive to COMMANDS list in install.sh. Add sed patterns for /revert and /archive. Update CLAUDE.md with new commands. Update README.md with new command docs.
- **Files**: install.sh, CLAUDE.md, README.md
- **Complexity**: Low
- **Depends on**: Steps 1-4

## Risks

| Risk | Severity | Mitigation |
|------|----------|------------|
| `git reset --soft` can lose work if user hasn't pushed | Medium | Warn user and confirm before resetting |
| `gh`/`glab` CLI might not be installed | Low | Fallback to asking user to create PR manually |
| Sequential /implement is long-running, context pressure | Medium | Commit after each step to save progress |
