# Implementation Plan: Install Re-install + Done Merge Flow

> Generated by /cc-plan on 2026-03-23

## Steps

### Step 1: Add config file and re-install support to install.sh

- **Description**: Save prefix to `.cc-workflow-config` after install. On re-install, read existing config, remove old command files, then install new ones.
- **Files**: install.sh
- **Complexity**: Medium
- **Depends on**: none

### Step 2: Add merge check to /done command

- **Description**: After verification and review, check if the feature branch is merged into the default branch. If not, ask user to merge via their platform. Wait for confirmation, then checkout default branch and pull.
- **Files**: commands/done.md
- **Complexity**: Medium
- **Depends on**: none

## Risks

| Risk | Severity | Mitigation |
|------|----------|------------|
| User's default branch could be main or master or develop | Medium | Detect via `git remote show origin` or check which exists |
| User might not have merged yet when they say "merged" | Low | Verify by checking if HEAD commit exists on default branch |
