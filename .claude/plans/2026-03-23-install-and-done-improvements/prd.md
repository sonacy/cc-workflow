# PRD: Install Re-install Support + Done Merge Flow

> Generated by /cc-plan on 2026-03-23

## Overview

Two improvements: (1) install.sh detects existing installations and handles prefix changes cleanly, (2) /done ensures code is merged to default branch before completing.

## User Stories

- As a user, I want to re-run install.sh to change my prefix without leaving stale command files behind.
- As a user, I want /done to remind me to merge my branch before wrapping up, so I don't forget.

## Acceptance Criteria

- [ ] install.sh reads `.cc-workflow-config` to detect existing prefix
- [ ] Re-install with different prefix removes old prefixed files
- [ ] Re-install with same prefix overwrites cleanly
- [ ] `.cc-workflow-config` records installed prefix for future re-installs
- [ ] /done checks if feature branch commits exist on default branch
- [ ] /done asks user to merge (via PR on any platform) if not merged
- [ ] /done checks out default branch and pulls after merge confirmed
- [ ] /done does NOT delete the feature branch (just switches away)

## Out of Scope

- Automatic merge (user handles via their platform's PR flow)
- Branch deletion
- Platform-specific PR creation commands
